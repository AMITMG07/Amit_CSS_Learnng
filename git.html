Git Codes Information

  1. Git Version :

    The command git --version is used to check the installed version of Git on your system. When you run this command in the terminal or command prompt, it outputs something like:git version 2.34.1

    2. This tells you which version of Git is currently installed. If Git is not installed, you'll likely see an error message indicating that the command is not found.

  2. Git Init:

    The git init command initializes a new Git repository in a directory. It can be used to start tracking a project with Git.

  3. Git Status:

    git status is a command used in Git to show the current state of your working directory and staging area. It helps you see which changes have been staged, which haven't, and which files aren't being tracked by Git.

  4. Gif add file:

    git add is a command used to stage changes in Git. This means that the specified file is added to the staging area, preparing it to be committed in the next git commit operation.

  5. Add message:

    git commit -m 'message' creates a commit with a message describing the changes. Make sure to stage files first using git add ..

  6. Setting Up Git for the First Time:

    To set up Git for the first time, you need to configure your user identity using the following commands:
    git config --global user.email "abc@gmail.com"
    git config --global user.name "Your Name"
    This ensures that your commits are associated with your name and email. The --global flag applies the settings to all repositories on your system. You can verify your configuration using git config --global --list. If you want to set these details for a specific repository instead, run the commands without the --global flag inside the repo folder.

  7. Adding All Files at Once:

    The git add . command stages all modified and new files in the current directory and its subdirectories for the next commit. It does not stage deleted files unless explicitly removed with git rm. This command is useful for quickly preparing all changes before running git commit.

  8. Git Log Information:

    The git log command displays the commit history of a Git repository. It shows details like commit hash, author, date, and commit message. By default, it lists commits in reverse chronological order, with the most recent commit first.

  9. Understanding git reset --hard "version code":

    The git reset --hard command is used to reset the current branch to a specific commit (), discarding all changes in the working directory and staging area. This means any modifications or uncommitted work will be lost. It is useful when you want to completely revert to a previous state in your repository. Be cautious while using this command, as it permanently deletes uncommitted changes.

  10. Git Reset --soft:

    The git reset --soft command is used to move the HEAD (current branch pointer) to a specific commit while keeping the changes staged in the index. This means all the changes from the commits after the specified one remain in the staging area, allowing you to commit them again if needed. It is useful when you want to undo commits but keep the changes for modification before recommitting. Unlike git reset --hard, it does not delete changes, making it a safer option when revising commit history.

  11. Git Remote Add Origin:

    The command git remote add origin https://github.com/AK/CSS_Learnng.git is used to link a local Git repository to a remote repository hosted on GitHub. Here, origin is the default name for the remote repository, and the given URL is its location. This connection allows you to push and pull changes between your local and remote repositories, enabling collaboration and backup. After adding the remote, you can use commands like git push -u origin main to upload your local code to GitHub.

  12. Understanding git branch -M main:

    The command git branch -M main is used in Git to rename the current branch to main. The -M flag stands for "move/rename" and forces the branch name change, even if a branch named main already exists. This command is particularly useful when initializing a new repository, as Git previously used master as the default branch. Many developers and organizations now prefer main as the default branch name for clarity and inclusivity. If you run this command in a new repository, it ensures that the initial branch is named main instead of master.

  13. Git Clone:

    Commnad git clone is used to copy a remote Git repository to your local machine. The basic syntax is git clone . You can also clone a specific branch using --branch or into a custom directory by specifying its name at the end.

  14. Git Pull:

    Command git pull is used to fetch and merge changes from a remote repository into the local branch. It combines git fetch (downloads updates) and git merge (integrates them) in one step, keeping your local branch up to date.
